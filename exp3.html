Aim

To understand and implement SQL aggregate functions (COUNT, SUM, AVG, MAX, MIN), GROUP BY, HAVING clauses, and learn about creating and managing views.

Theory

Aggregate functions perform calculations on a set of values and return a single value. Views are virtual tables based on the result set of an SQL statement.

Aggregate Functions

  • COUNT(): Counts number of rows
  • SUM(): Calculates sum of values
  • AVG(): Calculates average value
  • MAX(): Finds maximum value
  • MIN(): Finds minimum value

Views

  • Virtual tables
  • Based on SELECT queries
  • No physical data storage
  • Can simplify complex queries
  • Provides data security

Syntax Reference

Aggregate Functions


-- Basic Aggregate Function Syntax
COUNT([DISTINCT] column_name)
SUM([DISTINCT] column_name)
AVG([DISTINCT] column_name)
MAX(column_name)
MIN(column_name)

-- With GROUP BY
SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1;

-- With HAVING
SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1
HAVING aggregate_function(column2) operator value;

Views


-- Create View
CREATE [OR REPLACE] VIEW view_name [(column1, column2, ...)] AS
    SELECT column1, column2, ...
    FROM table_name
    WHERE condition;

-- Drop View
DROP VIEW [CASCADE | RESTRICT] view_name;

-- Update View
UPDATE view_name
SET column1 = value1
WHERE condition;

-- Create Complex View
CREATE VIEW complex_view AS
    SELECT t1.column1, t2.column2,
           aggregate_function(t1.column3)
    FROM table1 t1
    JOIN table2 t2 ON t1.id = t2.id
    GROUP BY t1.column1, t2.column2;

Important Notes:

  • Aggregate functions ignore NULL values
  • GROUP BY must include all non-aggregated columns in SELECT
  • HAVING filters groups, WHERE filters rows
  • Views can be simple or complex (with joins, aggregates)
  • Some views are updatable if they meet certain criteria

-- Basic Aggregate Functions
SELECT 
    COUNT(*) as total_orders,
    SUM(TOTAL_AMOUNT) as total_sales,
    AVG(TOTAL_AMOUNT) as avg_order_value,
    MAX(TOTAL_AMOUNT) as highest_order,
    MIN(TOTAL_AMOUNT) as lowest_order
FROM ORDERS;

-- GROUP BY with HAVING
SELECT 
    CUSTOMER_ID,
    COUNT(*) as order_count,
    SUM(TOTAL_AMOUNT) as total_spent
FROM ORDERS
GROUP BY CUSTOMER_ID
HAVING SUM(TOTAL_AMOUNT) > 2000;

-- Create a View
CREATE VIEW CUSTOMER_ORDERS AS
SELECT 
    O.CUSTOMER_ID,
    COUNT(*) as total_orders,
    SUM(O.TOTAL_AMOUNT) as total_amount,
    AVG(OI.QUANTITY) as avg_items_per_order
FROM ORDERS O
JOIN ORDER_ITEMS OI ON O.ORDER_ID = OI.ORDER_ID
GROUP BY O.CUSTOMER_ID;

-- Query the View
SELECT * FROM CUSTOMER_ORDERS
WHERE total_amount > 2000;

-- Drop the View
DROP VIEW CUSTOMER_ORDERS;
